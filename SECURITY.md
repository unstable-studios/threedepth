# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in ThreeDepth, please report it responsibly by emailing:

**contact@unstablestudios.com**

**Please do NOT open a public issue for security vulnerabilities.**

## What to Include

When reporting a vulnerability, please include:

- A clear description of the issue
- Steps to reproduce the vulnerability
- Potential impact of the vulnerability
- Any suggested fixes (if applicable)

## Response Timeline

We take security seriously and will respond to your report within **48 hours**.

We will work with you to:
1. Confirm the vulnerability
2. Determine the severity
3. Develop and test a fix
4. Release a patch
5. Credit you for the discovery (if desired)

## Supported Versions

We provide security updates for the latest release of ThreeDepth. Please ensure you're using the most recent version.

## Security Best Practices

ThreeDepth processes all 3D models client-side in your browser. No files are uploaded to our servers. However:

- Only load 3D models from trusted sources
- Be cautious when importing files from unknown origins
- Keep your browser up to date for the latest WebGL security patches

## Scope

Security issues related to:
- ✅ Application vulnerabilities
- ✅ Client-side code execution issues
- ✅ WebGL security concerns
- ✅ Dependency vulnerabilities

Out of scope:
- ❌ Third-party browser issues
- ❌ Social engineering attacks
- ❌ Physical access to user devices

Thank you for helping keep ThreeDepth secure!
